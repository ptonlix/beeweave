# BeeWeave

[English](README.md) | [中文](README-zh.md) | [Documentation](https://ptonlix.github.io/beeweave/) | [中文文档](https://ptonlix.github.io/beeweave/zh/)

BeeWeave is an **agent-native creation workbench** for building a data flywheel
around your creative process: collect source material, create with agents,
distill durable knowledge, reuse that context, and collect better material for
the next loop.

## Quick Start

Install BeeWeave and run setup in the workspace where you want runtime
`vault/` and `workbench/` folders:

```bash
pip install beeweave
bwe setup
```

Then use BeeWeave skills from your agent:

```text
/beeweave-ingest workbench/inbox
/beeweave-query what do I know about rate limiting?
/beeweave-update
```

## Core Commands

```bash
bwe info
bwe list
bwe setup --agents claude,codex
bwe setup --global-extra beeweave-capture
bwe uninstall
```

`bwe uninstall` removes BeeWeave-managed skills, bootstrap files, and
`~/.beeweave` config. It does not delete your `vault/` or `workbench/` content.

## Runtime Layout

`bwe setup` creates the generated workspace layout in the selected project:

```text
project/
+-- vault/
|   +-- concepts/
|   +-- entities/
|   +-- skills/
|   +-- references/
|   +-- synthesis/
|   +-- projects/
|   +-- _meta/
|   +-- _archives/
|   +-- _staging/
|   +-- .obsidian/
+-- workbench/
    +-- inbox/
    |   +-- captures/
    |   +-- web/
    |   +-- archived/
    |   +-- rejected/
    +-- articles/
    |   +-- drafts/
    |   +-- published/
    +-- library/
```

Use `workbench/inbox` for rough captures and imported sources. Use `vault/`
for stable knowledge that should be searched, linked, and reused.

## Named Vault Routing

Create named configs such as `~/.beeweave/config.work`, then route a single
request with `@name`:

```text
beeweave-query @work what do I know about deployment rollbacks?
@research update my BeeWeave vault
```

All supported agents can use this syntax, including Claude Code, Cursor, Windsurf, Codex, Gemini, Kiro, Hermes, OpenClaw, Pi, Copilot CLI, and generic `AGENTS.md` agents.

## Repository Layout

```text
beeweave/       # Python CLI and helpers
.skills/        # source skill definitions
bootstrap/      # user-project bootstrap templates
extensions/     # browser extension assets
tests/          # pytest suite
docs/           # MkDocs source documentation
```

Project-local setup can install bootstrap files such as `AGENTS.md`,
`CLAUDE.md`, `GEMINI.md`, and `HERMES.md`.

Runtime `vault/` and `workbench/` directories are generated by setup in the
user-selected workspace and should not be committed to this repository.
